# cv-creator LaTeX 参考流程

## 说明

这不是技能入口文件，而是面向现有 LaTeX 文件维护流程的参考文档。
当用户选择 `LaTeX` 或 `Markdown + LaTeX` 模板时，可参考本文档中的问答顺序、字段粒度和编译流程。

主技能入口请看根目录 [SKILL.md](../SKILL.md)。

---

## 目标

通过对话式问答，引导用户逐步填写简历信息。
每执行完一步，更新根目录下现有的 [resume.tex](../resume.tex)，并在需要时编译渲染 PDF。
工作内容描述采用 STAR 原则（Situation/Task -> Action -> Result）进行优化。

---

## 开场说明

开始时可向用户说明：

> 您好！我是 Brench 开发的简历助手，我将帮助您完成简历。您可以直接回答我的提问来填写信息，也可以把已有 PDF 简历放在项目根目录的 `cv/` 文件夹，我会先尝试读取并提取内容。

---

## 导入已有简历（可选）

在第一步之前，静默检查是否存在 `cv/` 目录且包含 PDF 文件：

```bash
ls cv/*.pdf 2>/dev/null || true
```

若存在 PDF 文件：

1. 使用 `/pdf` 技能读取 PDF 内容。
2. 解析提取以下信息：
   - 个人信息：姓名、邮箱、电话、籍贯、政治面貌、通讯地址
   - 教育背景：学校、专业、学历、时间
   - 实习经历：公司、岗位、时间、工作内容
   - 项目经历：项目名称、奖项、时间、工作内容
   - 学生工作：部门、岗位、时间、工作内容
   - 个人评价：英语水平、技术栈、爱好等
3. 向用户确认提取的信息是否准确。
4. 若有缺失或不确定内容，明确询问用户补充。

若不存在或读取失败，则继续正常流程。

---

## 第零步：环境检查

在进入正式问答前，静默检查 XeLaTeX 是否可用：

```bash
xelatex --version
```

若命令成功，可静默初始化宏包：

```bash
bash scripts/install-packages.sh 2>/dev/null || true
```

若命令失败：

1. 告知用户尚未安装 LaTeX 编译环境。
2. Windows 下可建议：
   ```bash
   winget install --id MiKTeX.MiKTeX -e --accept-source-agreements --accept-package-agreements
   ```
3. 安装后提示用户运行：
   ```bash
   bash scripts/install-packages.sh
   ```
4. 本轮停止，等待环境可用后继续。

---

## 第一步：个人信息

同时询问：

1. 姓名
2. 籍贯
3. 邮箱
4. 电话
5. 政治面貌
6. 通讯地址
7. 个人主页
8. 作品集链接

将这些信息映射到 [resume.tex](../resume.tex) 的头部个人信息区。
用户没有的字段直接省略，不保留空占位。

---

## 第二步：教育背景

询问用户有几段教育经历，然后逐一询问：

- 学校名称
- 专业
- 学历
- 时间段

更新教育部分内容。

---

## 第三步：照片（可选）

询问用户是否需要添加照片：

- 需要：提示将照片命名为 `you.jpg` 并放入约定位置，再启用照片相关命令
- 不需要：保持模板中照片相关内容关闭或注释

---

## 第四步：实习经历

逐一询问：

- 公司名称
- 岗位
- 实习时间
- 工作内容

处理方式：

1. 将工作内容精炼为 20 到 30 字的一句话。
2. 用 STAR 原则扩展为 2 到 3 点，每点 40 到 60 字。
3. 优先加入量化结果。
4. 时间倒序排列。

---

## 第五步：项目经历

逐一询问：

- 项目名称
- 奖项
- 项目时间
- 工作内容

处理方式与实习经历一致：

1. 先给一句概括。
2. 再扩成 2 到 3 个 STAR 要点。
3. 时间倒序排列。

---

## 第六步：学生工作

逐一询问：

- 工作部门
- 工作岗位
- 工作内容

建议用户填写 1 到 3 段即可，优先保留最能体现组织、协作和影响力的经历。

---

## 第七步：个人评价

询问：

- 英文水平
- 技术栈
- 爱好特长
- 其他补充

综合写成 3 到 5 点简洁评价，每点控制在 30 到 45 字。

---

## 完成后

1. 展示更新后的 [resume.tex](../resume.tex) 关键部分供用户确认。
2. 如需编译，执行：
   ```bash
   bash scripts/build.sh
   ```
3. 编译成功则告知用户 `resume.pdf` 已生成。
4. 编译失败则展示错误信息，并视情况建议先执行：
   ```bash
   bash scripts/install-packages.sh
   ```
